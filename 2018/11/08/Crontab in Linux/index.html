Crontab定时器的使用

date: 2018-11-08 01:34:37
tags: Linux

1. Install 安装

可执行以下命令查看是否已安装Crontab
crontab -l

若未安装，执行一下命令安装：

Ubuntu: apt-get install cron
Centos:  yum install vixie-cron

使用如下命令查看Crontab的执行状态：

service cron status

启动/关闭：

service cron start(restart) / close

2. Basic order 基础命令

crontab -l/e/u/r

-l 查看当前用户的定时任务
-u 切换用户
-e 编辑当前用户的定时任务
-r 删除定时任务
Exp： crontab -u user -e

3. Edit Timely Jobs 编辑定时任务

使用下列命令编辑定时任务

crontab -e

再编辑文件的下方新建一行代表一个定时任务:
任务格式：
* your command

上述五个星号分别代表：
0-59 - Minutes 分钟
0-23 - Hour 小时
1-31 - date 日期
1-12 - Month 月份
0-7 - Weekday 星期几

Eg:
_/2 service nginx restart 每隔2分钟重启nginx
0 23-7/1 service nginx restart 晚上23点到早上7点每隔1h重启
0,30 18-23 service nginx restart 每天18点-23点每隔30min重启
0-59/30 18-23 * service nginx restart 每天18点-23点每隔30min重启

4. Issues 遇到问题

Assert: 我们尽量采用Crontab调用Shell脚本的方式来执行所需任务，不要直接将命令行写道crontab的任务文件中。

**下面列举几个本人遇到的问题：
1 > Shell脚本执行顺畅，但crontab定是服务调用脚本不生效**
首先请查看日志，分析问题：

tail -f /var/log/cron                  查看cron的log日志
tail -n 200 /var/spool/mail/root       查看邮件服务器root用户日志的最后两百行
tail -n 200 /var/spool/mail/[userName] 查看User用户日志的最后两百行

请注意，在使用定时服务的时候最好默认所有相关的相关环境变量并没生效，请在脚本中source环境变量使之生效。
Eg: 

source ~/.bashrc 
source /etc/profile

若要使用浏览器等视图服务请在脚本中声明端口环境变量： 

export DISPLAY=:0
